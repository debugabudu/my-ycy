# ycy 后续规划（TODO）

分两块：**Agent 能力补强**（引擎与机制）与 **常用功能扩展**（生活与工作场景下的工具与 Skill）。实现时可交叉依赖（例如新工具要接进 `definitions` + `handlers`）。

## 一、Agent 能力补强

面向「更稳、更记得住、更快把经验变成 Skill」的底层能力，与具体业务场景解耦。

### 1.1 记忆

- [ ] **短期会话记忆**：持久化对话 `history`（含 run_id），支持 `--resume` 或按会话文件续聊，避免每次从零开始。
- [ ] **长期记忆（显式写入）**：仅通过专用工具（如 `memory_append` / `memory_search`）读写结构化片段，避免模型随口胡编「我记得」；支持按标签、时间检索。
- [ ] **可选向量检索**：对用户指定目录或「记忆库」建索引，查询时带引用片段，便于个人笔记与纪要。
- [ ] **压缩与锚点**：摘要策略可配置；压缩后保留用户设定「锚点」（人名、长期目标、禁忌等）不被冲掉。

### 1.2 安全与权限（严格）

- [ ] **工具级策略**：写文件、执行命令、联网、起子代理/队友、后台任务等分级（禁止 / 每次确认 / 允许）；可用环境变量 + 会话内开关组合。
- [ ] **路径与命令白名单**：工作区外默认禁止；`bash` 可配置允许前缀或封禁模式（在现有简单拦截之上增强）。
- [ ] **敏感数据**：日志、trace、导出内容中对密钥模式脱敏；文档说明「勿把 .env 交给模型全文读取」的默认建议。
- [ ] **子代理 / 队友隔离**：子代理按调用实例区分收件箱或会话 id，减少并行串扰；队友仅暴露 profile 允许的工具集（在现有合并规则上可再加硬上限）。

### 1.3 Skill 快速沉淀

- [ ] **从对话一键生成 Skill 草稿**：例如「把上一轮结论存成 Skill：xxx」→ 写入 `skills/<name>/SKILL.md` 模板（frontmatter + 正文），用户再人工微调。
- [ ] **Skill 版本与摘要**：frontmatter 增加 `version` / `updated`；`load_skill` 前可在系统侧展示一行摘要，减少重复加载全文。
- [ ] **脚手架命令**：`ycy init skill <id>` 生成标准 `SKILL.md` 与生活/办公示例段落。
- [ ] **与记忆联动**：可选「高频 Skill 片段」进入长期记忆索引，减少每次拉全量 Markdown。

### 1.4 可靠性与可观测

- [ ] **失败重试与超时**：对联网、子进程统一超时与可配置重试次数，结果结构化返回给模型。
- [ ] **用户可读的运行报告**：非程序员也能看懂「本轮调了哪些工具、是否成功」的一页摘要（可由 trace JSONL 生成）。

### 1.5 体验与交互（引擎层）

- [ ] **流式输出**：若网关支持，主循环对流式 token 友好展示。
- [ ] **简单 GUI 或 Web 壳**：保留 REPL，增加可选聊天窗口，降低日常使用门槛。

## 二、常用功能扩展（工具 + Skill）

在「生活与工作小助手」前提下，**工具**（`definitions` + `handlers`）负责可执行能力，**Skill** 负责流程、话术与领域模板；二者配合。

### 2.1 建议新增或封装的工具（示例方向）

| 方向 | 工具思路 | 说明 |
|------|----------|------|
| 日程与提醒 | `reminder_create` / `reminder_list`（落地为本地 JSON + 外部 cron/脚本触发） | 把「明天九点叫我」变成可查记录 |
| 日历 | `calendar_import` / `calendar_next_events`（ICS 或 API） | 与日程工具配合 |
| 笔记与检索 | `note_search`、`note_append`（约定 `notes/` 目录） | 第二大脑入口，减轻裸 `read_file` |
| 联网（受控） | `http_get` 或 `fetch_url` | 白名单域名、超时、仅 GET；天气、汇率、公开网页 |
| 剪藏/摘要 | `url_summarize`（依赖上项 + 模型摘要） | 文章读后摘要写入笔记 |
| 邮件（可选） | `mail_draft`（只生成草稿文件）或只读 `mail_list` | 先草稿后人工发送，降低风险 |
| 文件转换 | `pdf_text_extract`（封装常见库） | 报销单、说明书等生活场景 |
| 习惯打卡 | `habit_log` / `habit_status` | 本地 JSON 即可 |
| 模板填充 | `template_render`（变量表 + 模板路径） | 周报、请假条、邮件正文 |

（实现时按优先级选做；每加一个工具须同步 **definitions、handlers、README、TEST**。）

### 2.2 建议沉淀的 Skill（示例清单）

Skill 不写逻辑代码，主要给模型 **步骤、检查项、输出格式**；可与上表工具组合使用。

- [ ] **旅行 / 出差**：物品清单、行前核对、行程草稿结构。
- [ ] **会议纪要**：议题、决议、待办、跟进人四段式模板。
- [ ] **周报 / 月报**：本周完成、阻塞、下周计划、需协助事项。
- [ ] **家庭与财务**：月度账单分类说明、简单预算复盘话术（不替代专业记账软件）。
- [ ] **健康与节奏**：喝水/久坐提醒话术、与 `habit_log` 配合的每日一问。
- [ ] **学习与阅读**：读书笔记结构、一篇文章的「三点收获」摘要框架。
- [ ] **本地生活**：附近办事流程（依赖你所在城市与政策，Skill 内写通用提问清单）。

### 2.3 与现有能力的衔接

- **看板任务 `task_*`**：适合「跨多天的项目事项」；**`todo_write`**：适合当次会话内几步操作。
- **`load_skill`**：上述 Skill 就绪后，主会话与子代理（已默认并入）均可按需加载。
- **队友 `spawn_teammate`**：适合「长时间提醒、定时汇总 inbox」类生活向 profile（需在文档中给示例，避免仅强调编码）。

## 三、已实现能力的优化空间（可选）

下列为**当前代码里已有行为**，但存在体验、健壮性或边界上的改进余地。你可按需并入第一节/第二节，或单独排期；与 **§1.2、§1.4** 有重叠的已在括号标注。

### 3.1 子代理与队友运行时

- [ ] **子代理收件箱共用**：并行多个 `task` 时均使用 `subagent` 身份，消息可能串线（与 §1.2「子代理隔离」同源，可合并处理）。
- [ ] **队友循环魔法数**：`teammate._loop` 内单轮最多 50 次工具轮询等宜改为命名常量或环境变量，便于按模型速度调优。
- [ ] **队友对话中的英文占位**：如自动抢单后插入的 assistant 内容为英文，与中文系统提示不一致，可统一为中文或走配置。
- [ ] **空闲抢单策略**：`idle` 阶段扫 `.tasks` 取「第一个 pending」认领，多队友时无显式公平/锁，易抢同一条；可文档化现状或引入 claim 锁/排序规则。

### 3.2 协议与持久化

- [ ] **`plan_requests` / `shutdown_requests` 仅内存**：进程重启即丢；`plan_approval` 依赖他处写入 `plan_requests`，当前缺少队友侧标准「提交计划」工具流（与 §二 中协作类扩展可一起设计）。
- [ ] **收件箱与并发**：`read_inbox` 一次性读并清空文件；极端并发下需确认是否满足你的使用强度（一般单写者足够）。

### 3.3 后台任务与资源

- [ ] **`BackgroundManager.tasks` 只增不减**：长跑会话可能堆积；可增加 TTL、最大条数或 `clear_completed` 类工具。
- [ ] **通知丢失风险**：若主循环长时间未跑 `drain()`，队列理论上可积压（通常 REPL 场景问题不大）。

### 3.4 文件与命令安全（现状加强）

- [ ] **`bash` 拦截名单过粗**：仅靠子串匹配，易误杀或漏网；与 §1.2 合并做策略化规则更佳。
- [ ] **写文件无版本/备份**：`write_file` / `edit_file` 覆盖前可选 `.bak` 或写入 `.ycy/trash`，便于助手场景「说错就改」。

### 3.5 配置加载与校验

- [ ] **Agent / Skill 仅启动时加载**：改磁盘后必须重启进程；可增加 `/reload` 或 `reload_profiles` 工具（注意线程安全）。
- [ ] **未知工具名静默丢弃**：`filter_unknown_tools` 只打日志，profile 写错不易察觉；可选「严格模式：加载失败」或启动时汇总告警 UI。

### 3.6 主循环与上下文

- [ ] **`todo_write` 提醒频率**：多轮未更新待办会插入 reminder，对部分模型可能干扰节奏；可做开关或提高阈值。
- [ ] **lead 收件箱注入**：有消息时每轮并入 `history`，爆发式消息时 token 增长快；可合并摘要或限条数。
- [ ] **自动压缩触发点**：仅 token 阈值 + `compress` 工具，策略较简；与 §1.1「压缩与锚点」联动细化。

### 3.7 看板任务数据

- [ ] **任务 JSON 无版本字段**：未来若改结构，缺迁移钩子；可加 `schema_version` 或小迁移脚本。
- [ ] **`task_update` 与依赖图**：大量任务时全目录扫描更新 blockedBy，规模上去后可优化为索引。

### 3.8 可观测与终端体验

- [ ] **Trace / 日志默认偏技术**：面向「生活助手」用户时可提供「安静模式」：少打 debug、缩短工具日志预览。
- [ ] **REPL 无多行编辑/历史文件**：长期可结合 §1.5 图形壳或 `prompt_toolkit` 类增强。
